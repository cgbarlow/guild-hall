## Design Principles

These quests follow principles from the Guild Reference:

1. **Practical over theoretical** — Each quest produces tangible outputs
2. **Fail fast** — Iterative objectives allow early course correction
3. **Open source** — Several quests require sharing with the community
4. **NZ context** — Multiple quests address local needs and opportunities
5. **Skill progression** — Mix of Apprentice → Journeyman → Master
6. **Community building** — Quests encourage participation in events and mentorship
7. **Technical themes** — Direct alignment with GRASP, dreaming architecture, sovereignty pillars
8. **Guild values** — Balance of vibe coding creativity and engineering rigor
